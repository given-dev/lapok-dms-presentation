1. Purpose
This document lists all UI changes and additions required to align Given's frontend with the LAPOK DMS backend schema and real-world operations at the LAPOK Coca-Cola depot. It is the single source of truth for frontend adjustments before Phase 2 development begins.
The backend (PHP + MySQL) has been built to match LAPOK's actual workflow. Where the current UI diverges from that — in field names, data models, missing screens, or missing fields — this document describes exactly what needs to change.
2. Summary of Changes
Changes fall into four categories:
Role model — the driver/cadet distinction needs to be reflected in the system
Dispatch form — vehicle loading must capture both driver and cadet separately
Stock intake form — batch number and expiry date fields are required on delivery recording
Missing fields — EFRIS reference, cash handover confirmation, and customer credit balance
Missing pages — reports module, audit log view, user profile / change password

9. Notes
Location names (Gulu routes, customer addresses) will be updated in a separate pass — skip for now.
The current demo data (Kampala customers, KCA/KCB plates) can stay in the frontend until we wire up the real backend.
All API responses follow the format: { "success": true/false, "data": {...}, "error": "..." }.
Session is cookie-based. Frontend fetch calls need credentials: "include" in all requests.
EFRIS integration is a future phase — no backend work needed yet, just reserve the field and the nav slot.
Questions? Reach out on the LAPOK IT team group.
